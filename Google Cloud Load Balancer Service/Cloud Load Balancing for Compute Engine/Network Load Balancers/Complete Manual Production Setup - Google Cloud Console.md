🚀 Phase 1: Project SetupStep 1: Create a New ProjectA Google Cloud Project forms the foundational sandbox for all your resources. It tracks billing, permissions, and service limits.Navigation: ☰ Menu → IAM & Admin → Manage Resources → CREATE PROJECTOpen Google Cloud Console (Use an Incognito/Private window to avoid multi-account conflicts).Click the ☰ (hamburger menu) icon in the top-left corner.Navigate to IAM & Admin → Manage Resources.Click the CREATE PROJECT button at the top.Project Name: shopwave-prod (Google will auto-generate a unique Project ID below this).Location: No organization (or select your company's organization node).Click CREATE.Wait for the Notification: Look at the bell icon 🔔 in the top right. Once it spins and stops, click your new project from the top dropdown menu to ensure you are working inside it.Step 2: Enable Required APIsAPIs must be turned on to grant your project permission to use specific Google Cloud hardware and services.Navigation: ☰ Menu → APIs & Services → LibrarySearch for and click ENABLE on the following API cards:Compute Engine API: Allows you to create Virtual Machines (VMs) and networking.Cloud Load Balancing API: Required for creating Target Pools and Forwarding Rules.Cloud Monitoring API: Allows us to pull CPU and traffic metrics later for our dashboard.Note: Enabling APIs might take 1-2 minutes to initialize in the background.🔧 Phase 2: Network ConfigurationStep 3: Create a Custom VPC NetworkA Virtual Private Cloud (VPC) is your private slice of Google's network. We use "Custom" mode instead of "Auto" to have strict control over our IP ranges and security.Navigation: ☰ Menu → VPC Network → VPC Networks → CREATE VPC NETWORKName: shopwave-prod-vpcSubnet Creation Mode: Select Custom (Do not use Automatic, as it creates unnecessary subnets globally).Add Subnet:Name: shopwave-subnet-us-east1Region: us-east1 (South Carolina)IP Address Range: 10.0.0.0/24 (Provides 256 internal IP addresses, plenty for our web tier).Leave Private Google Access "Off" for this tutorial, and click DONE.Scroll down and click CREATE.Step 4: Configure Firewall RulesBy default, GCP blocks all incoming traffic. Later, we will attach this tag to our 3 VMs, securely binding the firewall only to them.🖥️ Phase 3: Web Server InstancesStep 5: Create Server Instances (www1, www2, www3)We will launch Linux VMs, install Apache, and write a custom startup script to automate the website creation.Navigation: ☰ Menu → Compute Engine → VM Instances → CREATE INSTANCEYou will create three identical servers. To ensure High Availability (HA), place them in different zones within the us-east1 region.Base Configuration (Apply to all):Region: us-east1 (South Carolina)Machine Configuration: Select General Purpose → E2 series → e2-medium (2 vCPU, 4GB RAM).Boot Disk: Click CHANGE.OS: Debian or Ubuntu (latest LTS version).Boot disk type: SSD persistent disk (For faster boot times).Size: 50 GB. Click SELECT.Networking: Expand Advanced Options → Networking.Network tags: Type shopwave-web and press Enter. (This links the firewall rules from Phase 2).Network interfaces: Ensure it is set to shopwave-prod-vpc and the shopwave-subnet-us-east1.Startup Script Setup:Under Advanced Options, click the Management tab (or Metadata depending on UI version).Look for Automation / Startup script.Paste the following bash script. This script runs automatically as root the very first time the VM boots. It updates the OS, installs Apache, writes our HTML page, and creates the health-check endpoint.Bash#!/bin/bash
# 1. Update OS and install Apache Web Server
apt-get update
apt-get upgrade -y
apt-get install apache2 -y
service apache2 restart

# 2. Create the main landing page
cat > /var/www/html/index.html << 'EOF'
<title>ShopWave - Server</title>
<style>
  body { font-family: Arial; background: #f0f2f5; text-align: center; padding: 50px; }
  .container { background: white; padding: 30px; border-radius: 10px; max-width: 600px; margin: 0 auto; }
  h1 { color: #1a73e8; }
  .server { background: #e8f0fe; padding: 10px; border-radius: 5px; }
  .status { color: #34a853; font-weight: bold; }
</style>
<div class="container">
  <h1>🏪 ShopWave Store</h1>
  <div class="server">
    <p>🖥️ Server: www1</p>
    <p>📍 Zone: us-east1-b</p>
  </div>
  <p class="status">✅ Status: Healthy</p>
  <p>Environment: Production</p>
</div>
EOF

# 3. Create the health check file
mkdir -p /var/www/html/health
echo "healthy" > /var/www/html/health/index.html

If it says "Site can't be reached", wait 2 more minutes for the startup script to finish running, then refresh.⚖️ Phase 4: Load Balancer ConfigurationStep 7: Reserve a Static IP AddressIf a server reboots, its temporary IP changes. We need a permanent, static frontend IP for our Load Balancer so our domain's DNS records never break.Navigation: ☰ Menu → VPC Network → IP Addresses → RESERVE EXTERNAL STATIC IP ADDRESSName: shopwave-lb-ipNetwork Tier: Premium (Routes traffic over Google's fast global network).IP Version: IPv4Type: Regional (Because we are building a Regional Network Load Balancer).Region: us-east1Attached to: Leave as None. Click RESERVE.📌 Copy the generated IP address (e.g., 34.120.x.x) to a notepad.Step 8: Create the Health CheckThe load balancer uses this to constantly ping our servers. If a server crashes, the health check fails, and traffic is instantly rerouted.Navigation: ☰ Menu → Network Services → Load Balancing → (Click Advanced Menu Toggle on the left) → Health Checks → CREATE HEALTH CHECKName: shopwave-health-checkProtocol: HTTP | Port: 80Request path: /health/ (This points to the specific folder we created in the startup script).Check interval: 5 seconds (How often it pings).Timeout: 5 seconds (How long to wait for a response).Healthy threshold: 2 (Requires 2 consecutive successful pings to mark a server "Good").Unhealthy threshold: 3 (Requires 3 consecutive failed pings to mark a server "Dead").Click CREATE.Step 9: Create the Target PoolThis is the logical bucket that holds our three servers together.Navigation: Load Balancing Advanced Menu → Target pools → CREATE TARGET POOLName: shopwave-target-poolRegion: us-east1Session Affinity: None. (If set to Client IP, a user is glued to one server. Setting it to None ensures true random/round-robin load balancing).Health check: Select shopwave-health-check from the dropdown.Instances: Click Add instances and check the boxes for www1, www2, and www3.Click CREATE.Step 10: Create the Forwarding RuleThe final glue. This takes our Static IP and binds it to the Target Pool.Navigation: Load Balancing Advanced Menu → Forwarding rules → CREATE FORWARDING RULEName: shopwave-forwarding-ruleRegion: us-east1IP address: Open the dropdown and select the static IP we reserved (shopwave-lb-ip).IP protocol: TCP | Port: 80Target: Select shopwave-target-pool.Click CREATE.✅ Phase 5: Testing & ValidationTest 1: Load DistributionOpen a new browser tab and navigate to http://[YOUR_STATIC_LOAD_BALANCER_IP]Hard-refresh the page multiple times (Ctrl+F5 or Cmd+Shift+R).Look at the webpage body. Wait 30-45 seconds, refresh the page, and www1 will automatically rejoin the rotation!📊 Phase 6: Monitoring SetupVisibility is crucial. Fix: Wait 3 minutes.Cause B: The shopwave-allow-http firewall rule is missing or has a typo. Fix: Verify it allows 0.0.0.0/0 on tcp:80.Cause C: Apache failed to install. Fix: Go to VM Instances, click www1, click View logs (Cloud Logging), and search for startup-script errors.Issue 2: Health Check Failing (Servers won't accept traffic)Diagnosis: Go to Load Balancing → Advanced Menu → Health Checks. Click your health check. If it shows 0/3 healthy instances, the load balancer is dropping all traffic.Cause: The Google Health Check IPs (35.191.0.0/16, 130.211.0.0/22) are blocked. Fix: Double-check shopwave-allow-health-check in VPC Firewall. Ensure you used commas between the IPs and applied the shopwave-web target tag.Issue 3: Refreshing the page doesn't change the server nameCause: Browser caching or Session Affinity is active.Fix: Use an Incognito window or hard-refresh. Go to Target Pools and ensure Session affinity is set to None, not Client IP.
